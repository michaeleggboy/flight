# Impeccable — FlightPath

Persistent design context for AI-assisted UI work. Update the **Design Context** section when product direction changes.

## Design Context

### Users

**Product**: FlightPath — multi-city flight route optimizer with a price ↔ time tradeoff (α) and ranked itineraries.

**Inferred audience**: People planning multi-stop trips who care about total cost, total time, and route ordering (round trip, multi-city, or flexible order).

**Typical context**: Desktop-style layout (fixed sidebar ~380px, main map/globe area, max content width ~1600px). Primary job: add cities, tune tradeoff, run optimization, compare results on the map.

*Confirm with the team*: leisure vs business, desktop priority, and whether “confidence” copy should sound more consumer-friendly

### Brand Personality

**Inferred from UI**: Modern, technical, premium — dark “cockpit” canvas with glass panels and vivid accent highlights (cyan / emerald / magenta).

**Voice (from current chrome)**: Short labels, uppercase section labels with tracking, concise actions (“Find Best Routes”).

**Emotional target (inferred)**: Competence and control over a complex decision — not playful or whimsical.

### Aesthetic Direction

- **Theme**: Dark-only in code today (`midnight` background, `slate-850` / `slate-750`, glass overlays). No light theme unless you add it.
- **Typography**: `DM Sans` as primary (`font-display` on `body`); `JetBrains Mono` available for numeric or code-like UI (`font-mono`).
- **Color system**: Prefer Tailwind extensions and CSS variables already defined — `accent.cyan`, `accent.magenta`, `accent.amber`, `accent.emerald`; avoid new one-off hex unless tokenized.
- **Surfaces**: `.glass` / `.glass-hover` for panels; borders typically `white/5`–`white/6` or accent-tinted when selected.
- **Motion**: Framer Motion on key actions; keep motion purposeful. Prefer `prefers-reduced-motion` for heavy or decorative animation when you add it.
- **Iconography**: Lucide, small sizes (often 14–18px), muted `text-white/30`–`text-white/40` for secondary affordances.

**Anti-patterns to avoid**: flat gray-on-gray without accent hierarchy; random spacing outside Tailwind scale; removing focus rings on interactive controls.

### Design Principles

1. **Token-first styling** — extend the existing palette (`midnight`, `slate-`*, `accent-`*) and `:root` variables in `frontend/src/styles/index.css` rather than scattering raw colors.
2. **Glass for chrome, map for story** — controls and results stay in frosted panels; the globe/map remains the visual anchor.
3. **One clear primary action** — high-contrast cyan→emerald gradient CTA; secondary actions stay subdued until relevant.
4. **Scannable hierarchy** — uppercase section labels, consistent `space-y-`* / `gap-`*, and icon+label pairs for dense fields.
5. **Accessible by default** — sufficient contrast on text and controls; visible keyboard focus; don’t rely on color alone for critical distinctions (pair with labels/icons).
